# HANDOVER

日期：2026-07-19

## 本次完成

- 修复微信真机扫描裁切结果全黑：移除原图阶段的九参数 `CanvasContext.drawImage`，改用兼容性更稳定的五参数绘制，将原图平移到纸张包围区域内再读取像素。
- 不再把整张高分辨率照片读入 JavaScript。根据低分辨率检测到的四角计算原图纸张包围框，只处理纸张及少量安全边距，减少内存、像素读取量和透视计算量。
- 透视循环改为按行递增单应矩阵分子与分母，不再为每个像素创建坐标对象和调用多层函数；继续保留双线性插值及分片让出主线程。
- 增加原图缓冲区、透视结果和画布写回结果三阶段有效像素校验，任何阶段出现透明/全黑数据都会停止导出并提示重试，不再保存黑色文件。
- 修复条纹竹席背景样图的纸张识别：亮度候选若上下边或左右边收缩比例异常，会切换到 Canny 四边结果，避免把纸张错误识别为尖角梯形。
- 用户本次提供的实拍图已加入 `printed-letter-on-bamboo-mat.jpg` 回归夹具，自动裁切完整保留纸张并排除竹席、床垫等背景。

## 真机复验重点

- 对同一张竹席背景文档连续拍摄和裁切 5 次，确认每次预览均显示文档而不是黑图。
- 比较本版本与上一版本从拍照到裁切完成的时间，确认纸张占照片一半左右时处理时间明显下降。
- 连续扫描 3 至 5 页，确认后台裁切期间仍可继续拍照，完成后无黑页且 PDF 页序正确。
- 在纸张接近画面边缘以及检测失败回退整图两种情况下复验，确认不会因包围框计算丢失正文。

## 已知边界

- 原始相片已经失焦或抖动时，裁切不会虚构不存在的文字细节。
- 纸张四边大面积遮挡或与背景完全同色时仍会回退完整照片，避免错误裁掉正文。
- 微信旧 Canvas 的实际性能与设备内存有关，最低配置目标 iPhone 仍需完成连续多页真机压力测试。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
