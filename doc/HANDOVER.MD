## 本次完成

- 导出文件页不再在进入页面或切换格式时自动生成文件，只有点击“导出文件”或“分享好友”才开始生成当前格式。
- 导出完成后复用同格式成品，避免再次编码；切换格式会使旧格式缓存失效，确保分享内容与当前选择一致。
- 微信要求分享接口必须由用户点击同步触发。未生成文件时首次点击“分享好友”负责生成，完成后按钮变为“发送给好友”，再次点击直接打开微信分享面板，不再显示确认弹框。
- 关于页新增与帮助反馈页一致的公众号二维码、反馈说明和联系邮箱，并保留备案信息。
- 增加生产包回归检查，禁止旧的页面进入即自动准备分享逻辑重新出现。

## 上线复验重点

- JPEG/PNG/PDF 各执行一次“导出文件”，确认生成、打开和文件名正确。
- 每种格式首次点击“分享好友”应生成文件并显示“发送给好友”；第二次点击应直接打开微信分享面板。
- 切换导出格式后必须重新生成，不能分享上一个格式。
- 关于页和帮助与反馈页的二维码应可长按识别，邮箱可复制，备案号可打开工信部页面。

## 已知平台边界

- 微信不允许异步生成完成后自动调用文件分享接口，因此在禁止页面自动生成的前提下，首次分享点击用于生成、第二次点击用于发送是平台约束下的可靠流程。
- 大体积多页 PDF 的生成耗时取决于页数、图片尺寸和设备性能，生成期间按钮会显示处理中并阻止重复任务。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
