# HANDOVER

日期：2026-07-19

## 本次完成

- 修复微信真机扫描页持续显示“重新裁切”的误判：删除 `canvasPutImageData` 后立即读取像素的校验。旧 Canvas 的写入成功回调早于实际刷新，立即回读可能得到空缓冲区。
- 写入裁切像素后增加 24ms 刷新等待，再调用 `canvasToTempFilePath`，避免导出时序竞争。
- 原分辨率读取增加两级兼容路径：优先仅读取纸张包围区域；旧 iOS Canvas 不支持负偏移绘制时，自动改为读取完整原图并使用原图坐标透视裁切。
- 保留 JavaScript 原图缓冲区和透视结果两层有效像素校验，真正的透明或全黑数据仍不会进入 JPEG 导出。
- 扫描预览页在失败卡片中显示具体错误原因，同时保留单页“重新裁切”，不再只有无法诊断的统一状态。
- 新增旧 Canvas 局部读取为空、完整原图回退成功的单元测试。

## 真机复验重点

- 对同一文档连续拍摄 5 次，确认预览从“裁切中”进入“裁切完成”，不再统一显示“重新裁切”。
- 在旧 iPhone 上拍摄纸张靠近边缘的照片，确认局部读取失败时完整原图回退仍能生成清晰裁切图。
- 连续扫描 3 至 5 页，确认后台裁切期间可继续拍照，所有页面完成后可生成并预览 PDF。
- 若仍有单页失败，记录卡片上新增的具体错误文案，区分原图读取、透视结果和临时文件导出问题。

## 已知边界

- 原始照片失焦、抖动或纸张四边被大面积遮挡时，客户端无法恢复不存在的细节。
- 完整原图回退只在局部 Canvas 读取为空时启用，低内存旧设备上的处理时间可能比正常路径略长。
- 微信旧 Canvas 的实际性能与设备内存有关，发布前仍需完成最低支持机型的连续多页压力测试。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
