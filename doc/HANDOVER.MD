# HANDOVER

日期：2026-07-20

## 本次完成

- 扫描流程改为手动取景框裁切：用户将纸张四边对齐竖版 A4 辅助框后拍照，不再执行自动边沿识别和透视校正。
- 拍照前读取相机区域与辅助框的真实屏幕坐标，并按相机 `aspect-fill` 显示关系映射到原始照片像素，避免状态栏、底部拍照区和不同机型比例造成裁切偏移。
- 微信端只执行一次原图平移绘制，裁切画布与输出图片均使用映射后的原始像素宽高；不缩放、不透视、不插值，JPEG 质量为 1。
- H5 开发预览补充同等的一比一源像素裁切路径，连续扫描、预览、排序和进入签字页保持可用。
- 微信生产包移除扫描页已停用的自动识边离屏读取代码，减少该页面运行代码和处理时间。

## 真机复验重点

- 在不同刘海屏和屏幕比例设备上，将 A4 纸四边贴近辅助框，确认结果与框内画面一致。
- 放大对比原照片和裁切结果的小字号笔画；裁切结果不应出现缩放或透视插值造成的新增模糊。
- 连续拍摄 3 至 5 页，确认拍照后快速进入预览，后台裁切不会阻塞继续扫描。
- 检查非常高分辨率照片在低内存设备上的画布导出稳定性。

## 已知边界

- 本版按固定辅助框裁切，纸张未与辅助框对齐时不会自动纠正倾斜或透视。
- JPEG 裁切需经过微信 Canvas 解码和质量 1 重编码，因此文件二进制不会与原 JPEG 完全相同；本版保证不缩放、不透视并保持源像素尺寸，避免额外的几何采样损失。
- 相机自身失焦、抖动和曝光造成的模糊无法由裁切恢复。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
