# HANDOVER

日期：2026-07-19

## 本次完成

- 根治真机扫描持续出现“读取原图失败”：不再使用隐藏的旧版组件 Canvas 二次读取数千像素相机原图。
- 原分辨率路径改用微信 2D 离屏 Canvas。图片节点完成解码后，仅绘制并读取检测到的纸张包围区域，再执行原分辨率透视裁切。
- 离屏图片加载设置 1.5 秒上限；设备不支持离屏 Canvas、内存不足或图片节点失败时，自动使用已经成功读取并验证的 900px 检测缓冲继续裁切，不再把整页标记失败。
- 扫描处理 Canvas 的 CSS 尺寸与像素尺寸同步，删除生产包中将画布强制压成 `1px × 1px` 的覆盖样式。
- 保留检测缓冲和透视结果的有效像素校验，真正的全黑/透明数据仍不会被导出。
- 结果记录 `scanResolutionMode`，可区分原分辨率输出和设备兼容输出。

## 验证结果

- 文案合规、敏感信息检查和类型检查通过。
- 单元测试：24 个文件，104/104 通过；覆盖离屏原图、黑图拦截和离屏不可用自动降级。
- H5 E2E：25/25 通过；实拍文档扰动裁切成功率保持 90% 以上。
- H5 与微信生产构建通过。
- 微信生产包静态冒烟通过，确认包含离屏原图读取、兼容降级和动态 Canvas 尺寸，且不包含旧的“读取原图失败”致命分支。
- 最新生产包位于 `dist/build/mp-weixin`。

## 真机复验重点

- 连续拍摄同一份文档 5 次，每页应从“正在裁切”进入“裁切完成”，不再出现“读取原图失败”。
- 查看清晰度：正常设备应使用原分辨率离屏裁切；只有离屏能力或内存异常时才进入 900px 兼容输出。
- 连续扫描 3 至 5 页，确认后台裁切期间可继续拍照，完成后 PDF 页序和预览均正常。
- 低内存设备应优先保证成功生成兼容裁切，不应卡死、黑图或要求重新拍摄。

## 已知边界

- 原始照片已经失焦、抖动或纸张边缘被大面积遮挡时，客户端无法恢复不存在的文字和边缘细节。
- 极低内存设备可能使用 900px 兼容结果；这是防止扫描完全失败的最后保障，正常基础库优先使用原分辨率。
- 本机微信开发者工具存在多个运行实例，自动化 CLI 启动测试发生连接超时；生产包静态冒烟已通过，远端 CI 将在干净环境复验完整流程。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
