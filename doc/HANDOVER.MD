## 本次完成

- 扫描裁边引入 JSFeat 成熟计算机视觉能力，采用高斯降噪、Canny 边缘、方向直线拟合、四边交点和几何可信度约束识别纸张，替换仅依赖亮度区域的复杂背景路径。
- 裁切改为标准四点单应性透视变换，不再用四边线性插值近似；像素采样由最近邻改为双线性插值，降低锯齿、断字和局部块状模糊。
- 彻底拆分识别分辨率与导出分辨率：900px 预览只用于快速找边，最终重新读取相机原图并把四角坐标映射回原图执行裁切，不再将 900px 缩略图作为成品。
- 扫描页隐藏画布支持按检测图、相机原图和最终裁切结果动态调整像素尺寸，JPEG 输出质量从 0.90 提升到 0.98。
- 用户提供的公文实拍图加入回归夹具。1280×1707 原图最终生成约 1177×1234 的透视成品；旧链路只能生成约 661×693。
- 新增光照、对比度和传感器噪声扰动回归，20/20 次准确定位纸张，自动化成功率 100%，高于 90% 验收线。
- 保留原折叠纸、反光电脑外壳和键盘背景回归，确保新算法不会只适配单张样图。

## 已知边界

- 算法不会虚构相机原图中已经失焦或运动模糊的细节；本次保证的是裁切流程不再额外降分辨率。
- 当纸张四边大面积被遮挡，或纸张与背景完全同色且无可见边缘时，会回退完整照片以避免错误裁掉正文。
- 超高像素照片的最终透视处理需要较多内存，仍需在目标最低配置 iPhone 上完成连续多页真机压力复验。
- Sass 构建仍有上游 legacy API 与 `@import` 弃用警告，不影响当前生产包。
