# 签字大师本地持久化实施

日期：2026-07-11

## 存储分层

微信小程序使用 `wx.env.USER_DATA_PATH/sign-master` 作为受管目录：

```text
sign-master/
  state/
    signatures.json
    templates.json
    files.json
    active-project.json
    identity.json
    security-records.json
  signatures/
    signature-*.png
  documents/
    picked-*.*
  exports/
    *.signed.pdf
    *.signed.png
    *.signed.jpg
```

- `Storage` 保留状态索引和设置，用于快速读取。
- 签名、模板、文件索引、当前工程、鉴别身份和溯源记录同时写入 `state/*.json`。
- 应用启动时优先读取 JSON；若 Storage 被清空，会用 JSON 自动恢复索引。
- 旧版本只有 Storage 数据时，首次读取会自动写入 JSON，完成无感迁移。
- 签名图片、导入文档和导出文件按类型复制或写入受管目录，不再依赖临时路径。
- 删除签名或已签署文件时同步删除对应受管文件。

## 平台降级

- 微信小程序：JSON 文件 + Storage 双写，文件进入 `USER_DATA_PATH`。
- H5：继续使用 `localStorage` 和浏览器 Blob，保证开发与自动化测试可运行。
- 无文件系统能力的平台：回退到 `uni.saveFile`，失败时保留原路径并明确由上层处理失效状态。

## 验证

- `tests/unit/local-repository.test.js` 模拟清空 Storage 后从 `USER_DATA_PATH` 恢复模板。
- `tests/unit/source-picker.test.js` 验证签名文件复制到受管目录。
- 文件名使用时间和随机后缀，避免多页相册文件在同一毫秒内互相覆盖。

## 数据边界

`USER_DATA_PATH` 属于当前微信小程序的本地用户数据。卸载小程序或在微信中执行“清除数据”仍会删除这些文件；如需跨设备或卸载后恢复，必须增加用户知情同意的云端备份能力。
