# 签字大师 1.0 开发日志

## 2026-07-19 - 微信原图读取失败根治

- 查明真机失败原因：处理 Canvas 的像素属性虽动态放大，但生产 WXSS 最终将显示面压成 `1px × 1px`；旧组件 Canvas 在读取高分辨率相机原图时返回透明缓冲区。
- 原分辨率读取迁移到微信 2D 离屏 Canvas，等待本地图片节点解码后仅读取纸张包围区域，继续保留原分辨率透视输出。
- 离屏读取增加 1.5 秒上限和 900px 已验证缓冲降级路径；原图二次读取失败不再终止扫描，也不再显示“读取原图失败”。
- 处理 Canvas 的 CSS 尺寸与像素尺寸同步，并移至视口外，不再使用 1px 隐藏画布。
- 新增离屏原分辨率、黑图拦截、离屏不可用仍完成扫描及微信编译产物回归检查。
- 当前 104/104 单元测试、25/25 H5 E2E、H5/微信生产构建和微信产物静态冒烟通过。

## 2026-07-19 - 真机持续重试误判修复

- 删除微信旧 Canvas 在 `canvasPutImageData` 成功后立即 `canvasGetImageData` 的不可靠回读校验，避免画布尚未刷新时被统一判定为裁切失败。
- 裁切像素写入后等待短暂刷新再导出，原图缓冲区和透视结果的有效像素校验继续保留。
- 原图纸张包围区域读取为空时自动回退完整原图读取，并将四角坐标切换到原图空间后继续透视裁切。
- 扫描失败卡片显示具体错误原因，便于真机区分读取、透视和导出阶段问题。
- 新增旧 iOS Canvas 负偏移读取失败的兼容回归；当前 104/104 单元测试、25/25 H5 E2E、H5/微信生产构建与微信产物冒烟全部通过。

## 2026-07-19 - 微信黑色扫描修复与透视性能优化

- 修复微信旧 Canvas 上九参数 `drawImage` 读取原图可能得到透明黑缓冲区的问题，改为五参数原图平移绘制，并只读取检测纸张的原分辨率包围区域。
- 透视矫正改为按行增量计算单应矩阵，移除逐像素坐标对象和函数调用；保留原分辨率、双线性插值与异步分片。
- 原图读取、透视结果和画布写回均增加有效像素校验，全黑数据不再进入 JPEG 导出。
- 亮度候选增加对边长度一致性校验，竹席背景实拍图会拒绝尖角错误候选并使用 Canny 四边结果。
- 新增竹席背景实拍回归和黑缓冲区禁止导出单测；当前 103/103 单元测试、25/25 H5 E2E、H5/微信生产构建、微信产物冒烟与生产依赖审计全部通过。
