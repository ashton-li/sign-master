## 指令：构建「签字大师」（SignMaster）小程序 1.0 版本

请基于 **uni-app + Vue 3 + Tailwind CSS + Pinia** 技术栈，创建项目 `signpen`，实现以下四个主页面和所有交互。项目需完全遵循「光之笔触·流光溢彩」设计语言，包含动态流光背景、玻璃拟态卡片、脉冲按钮及自动/手动主题切换。

### 1. 技术栈与工具
- **框架**：uni-app 3.x（Vue 3 Composition API + `<script setup>`）
- **样式**：Tailwind CSS 3.4（自定义配置见下文）、SCSS
- **状态管理**：Pinia 2.x
- **小程序适配**：weapp-tailwindcss（处理小程序不支持的部分 CSS）
- **构建**：Vite（uni-app 内置）
- **代码风格**：2 空格缩进，组件 PascalCase，事件 `handle` 前缀

### 3. 核心功能与页面要求

#### 3.1 首页（`pages/home/index`）
- 顶部品牌区域：渐变动画图标（笔+纸+签名元素），标题“签字大师”，副标题“SIGNPEN · 光之笔触”。
- 区域标题“已标注文件”及数量角标（例如 3 份）。
- 文件列表卡片（使用 `FileCard` 组件）：
  - 左侧彩色垂直光条（3px，`bg-brand`，带发光阴影）
  - 缩略图（圆角，带高光伪元素）
  - 文件名称、日期、签名数量
  - 状态标签“已标注”，含呼吸圆点动画
  - 右侧箭头 `›`
- 空状态提示：浮动图标 + 引导文案“点击下方 ✦ 签署 ✦ 按钮”。
- 点击卡片时调用 `useToast` 显示“查看文件详情”。

#### 3.2 模板页（`pages/templates/index`）
- 双列网格展示模板卡片（使用 `TemplateCard` 组件）。
- 每张卡片：
  - 上部预览区（模拟文档背景），内含 1~2 个虚线签名位指示器（带脉冲动画）。
  - 模板名称、签名位数量。
- 点击卡片时 Toast 提示“应用模板”。

#### 3.3 签名管理页（`pages/signatures/index`）
- 双列网格展示签名卡片（使用 `SignatureCard` 组件）。
- 每个卡片：
  - 手写体签名预览（Caveat/楷体风格，字号可变化）。
  - 标签文字（如“默认”“常用”）。
  - 最后一个卡片为“添加签名”占位，显示“+”。
- 点击签名卡片 Toast 提示“已选择：XXX”，点击添加卡片提示“新建签名”。

#### 3.4 设置页（`pages/settings/index`）
- 分组圆角容器，包含：
  - **深色模式**：左侧图标 🌗 + 文字，右侧 `ThemeToggle` 开关组件。
  - **关于**：显示“Signature Pro v2.0”。
  - **通知**：显示“已开启”。
- 点击深色模式行，循环切换主题（跟随系统 → 深色 → 浅色 → 跟随系统），并 Toast 提示当前模式。

### 4. 设计规范与视觉元素

#### 4.1 色彩（亮色 / 暗色）
使用 CSS 变量，在 `tailwind.config.js` 中扩展：
- `brand`：`#5856e0`（亮），`#7c7aff`（暗）
- `surface`：`#ffffff`（亮），`#151624`（暗）
- `ink`：`#1a1c26`（亮），`#eeeff5`（暗）等
- 阴影包含内高光 `inset 0 1px 0 rgba(255,255,255,0.7)`（亮色模式）

#### 4.2 玻璃卡片 (`.glass-card`)
- 背景：`rgba(255,255,255,0.65)` + `backdrop-filter: blur(22px) saturate(150%)`
- 边框：`rgba(0,0,0,0.06)`
- 顶部高光线：`::after` 伪元素，渐变白色，透明度变化
- 按压态：`scale(0.975)`，阴影加深
- 霓虹变体：`.neon-active` 添加外发光边框

#### 4.3 脉冲按钮 (`.pulse-btn`)
- 背景 `brand`，白色文字，圆角 `20px`
- 双伪元素实现两层扩散脉冲光环（动画 `halo-pulse`，2.5s）
- 按压缩放至 0.9

#### 4.4 动态背景（仅 H5 环境渲染）
- 三个大光斑（`orb-1/2/3`）缓慢漂移（动画 `orb-float` 14~20s）
- 八个小粒子从底部上升（动画 `particle-rise` 7~12s）
- 小程序环境直接隐藏，使用纯色背景

#### 4.5 动画定义
在 `tailwind.config.js` 的 `keyframes` 中定义：
- `orbFloat`、`particleRise`、`haloPulse`、`statusBreathe`、`pageIn`
- 页面切换动画使用 `pageIn`（模糊淡入 + 弹性位移）

### 5. 交互与状态管理

#### 5.1 底部导航 (`BottomNav.vue`)
- 固定底部，毛玻璃背景
- 四个常规 Tab（文件、模板、签名、我的）+ 中间“签署”按钮（使用 `PulseButton`）
- 点击 Tab 调用 `uni.switchTab` 切换页面
- 点击“签署”按钮切换至首页并 Toast“开始新建标注流程”（后续可扩展为实际签署流程）
- 当前激活 Tab 高亮，图标带发光投影

#### 5.2 主题切换 (`stores/theme.js`)
- 状态：`mode` (auto/light/dark)，`systemDark` (boolean)
- 初始化时读取 `uni.getStorageSync('theme-mode')`，监听系统主题变化
- `toggle()` 循环：auto → dark → light → auto，并持久化
- 通过 Pinia 在 `App.vue` 中给根 `view` 动态添加 `theme-light` 或 `theme-dark` 类

#### 5.3 Toast 提示 (`composables/useToast.js`)
- 响应式变量控制显隐，自动 1.8 秒消失
- 页面组件中调用 `show('消息')`

#### 5.4 数据模拟
- 文件、模板、签名数据均以 `ref` 数组存放于各自页面组件内（或 store 中），暂不涉及 API 请求。

### 6. 小程序适配要点
- **毛玻璃降级**：小程序不支持 `backdrop-filter`，使用 `#ifdef MP-WEIXIN` 条件编译，将玻璃卡片背景降级为纯色半透明（例如 `rgba(255,255,255,0.8)`），并去除 `backdrop-filter`。
- **动态背景移除**：`AmbientBackground` 组件整体使用 `#ifdef H5` 包裹，小程序不渲染。
- **自定义 TabBar**：`pages.json` 中 `tabBar.custom` 设为 `true`，并确保每个页面都引入 `BottomNav` 组件。
- **Tailwind 适配**：使用 `weapp-tailwindcss` 插件处理伪类、`backdrop-filter` 等不兼容项，配置 `corePlugins.preflight: false`。
