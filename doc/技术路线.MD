## 核心技术亮点与高门槛实现

以下列举了「签字大师」在工程化实现过程中引入的**前沿技术方案**，确保产品在性能、体验与技术壁垒上领先同类竞品。

---

### 1. 自研多层合成渲染引擎
- **技术方案**：基于 `OffscreenCanvas` 构建独立的**图层管理器**，每个签名/标注作为独立图层，包含变换矩阵（位移、缩放、旋转）、透明度、混合模式等属性。主画布仅做最终合成，避免重复绘制。
- **高门槛体现**：
  - 实现**脏矩形优化**：仅重绘发生变化的图层区域，提升复杂文档的渲染帧率。
  - 采用**离屏缓存策略**：为每个图层维护独立的位图缓存，只在图层属性变更时重新光栅化，普通拖拽直接 `drawImage`。
  - 支持图层顺序调整、隐藏/锁定等完整图层操作栈。

### 2. 超轻量 AI 签名抠图引擎
- **技术方案**：将轻量级语义分割模型（如 `PP-HumanSeg` 或定制 `MobileNetV3-Small`）转换为 `ONNX` 格式，通过 `WebAssembly` + `ONNX Runtime` 在客户端执行推理，实现**无需服务端**的签名笔迹提取。
- **高门槛体现**：
  - 模型大小控制在 2MB 以内，加载时间 < 300ms。
  - 使用 `Canvas` 获取图像数据，预处理（归一化、缩放）后喂入模型，后处理采用高斯模糊羽化边缘，实现像素级透明签名。
  - 微信小程序内利用 `WXWebAssembly` 支持，或降级为 HSV 色彩空间自适应阈值抠图，确保多端可用。

### 3. 全场景手势统一解调器
- **技术方案**：设计**有限状态机（FSM）** 统一处理所有触摸事件，解决单指绘制、双指缩放旋转、单指拖拽、长按锁定等操作的冲突。
- **高门槛体现**：
  - 状态转移条件包含：触点数量、移动距离、时间间隔、当前激活工具模式。
  - 内置**手势预测与防抖**：通过速度向量判断用户意图，避免误触发。
  - 支持 iPad 端外接键盘方向键微调，以及 Apple Pencil 的压感与倾角数据读取（`Pointer Events`）。

### 4. 实时笔迹平滑与压感模拟
- **技术方案**：采用**二次贝塞尔曲线**实时拟合触摸轨迹，结合速度自适应笔宽变化（速度越快笔触越细）模拟真实书写压力。
- **高门槛体现**：
  - 使用 **Pointer Events** 获取原生压感值（`pressure`），在小程序不支持的场景下利用**速度映射算法**产生伪压感。
  - 笔迹数据以 `point` 数组实时存储，支持**增量式 SVG 路径导出**，便于后续生成矢量签名。

### 5. 智能签名位检测（传统 CV + AI 可选）
- **技术方案**：对上传的文档图片进行预处理（灰度化、自适应阈值、轮廓检测），通过**霍夫变换**或**模板匹配**定位“签名/盖章”等文字区域，自动标注为候选签名位。
- **高门槛体现**：
  - 使用 `OpenCV.js`（编译为 WebAssembly）在客户端执行完整视觉流程，无需网络请求。
  - 结合轻量级 OCR（如 `Tesseract.js`）识别关键词，提高定位准确率。
  - 对识别结果进行**非极大值抑制（NMS）**，合并邻近区域，输出合理的签名位坐标。

### 6. 高保真 PDF 与矢量导出
- **技术方案**：通过 `jsPDF` 结合 Canvas 绘制结果生成含矢量元素的 PDF 文件，支持嵌入透明签名、可编辑文字。
- **高门槛体现**：
  - 将签名笔迹数据转换为 SVG 路径，直接写入 PDF，保持无限缩放清晰度。
  - 支持**PDF 表单域**生成，可在导出文件中保留可编辑的签名位置。

### 7. 极致性能的内存与渲染优化
- **技术方案**：签名画布采用**双缓冲机制**（前台 Canvas 展示，后台 Canvas 实时绘制），避免画面闪烁。
- **高门槛体现**：
  - 针对大尺寸文档（如 A4 300dpi）进行**四叉树分割**，按需加载可见区域，支持流畅缩放与滚动。
  - 使用 `requestAnimationFrame` 统一调度渲染任务，合并多次状态更新为一次绘制。
  - 监控纹理内存使用，自动释放未激活图层的位图缓存，防止崩溃。

### 8. 跨平台一致性的主题与动效系统
- **技术方案**：基于 CSS 自定义属性（`--custom`）构建**设计令牌**，通过 Pinia 状态管理实现自动/手动主题切换，所有组件统一消费令牌。
- **高门槛体现**：
  - 暗色模式下自动调整阴影、透明度、毛玻璃效果（`backdrop-filter`降级方案）。
  - 动画曲线统一采用 Apple 同款 `cubic-bezier`，并通过 `will-change` 与 GPU 加速确保 60fps。
  - 小程序端通过 `weapp-tailwindcss` 编译时兼容处理不支持的 CSS 特性。

### 9. 离线优先与增量同步架构
- **技术方案**：利用 `IndexedDB`（或微信 `FileSystemManager`）存储所有签名素材、模板和项目文件，所有操作无网络也能完成。
- **高门槛体现**：
  - 实现**增量快照**：每次编辑仅保存变更图层的数据，而非全量画布，历史记录栈占用极小。
  - 结合 `Web Worker` 进行后台图像处理与导出，保证主线程 UI 流畅。
  - 预留云端同步接口，支持后续多设备协作。
